## Scope

This baseline captures the repository contracts for the five dogfooding scenarios declared in `fixtures/dogfooding/scenarios.json`.

The local scripts scaffold and validate golden-run artifacts, but they do not invoke Claude Code skills by themselves. Therefore this baseline is marked `local-contract-replay`: the traces and result contracts are authored from the checked-in skill contracts and fixture manifest, then validated by `scripts/dogfood-report.sh`.

## Live Capture Attempts

Two Claude Code live-capture attempts were made on 2026-04-22 using `claude -p --plugin-dir plugins/claude-tech-squad` with repository editing tools disabled.

| Scenario | Result | Evidence |
|---|---|---|
| layered-monolith | completed as inline demo | Claude Code emitted discovery trace lines and a blueprint, but did not emit a `result_contract` block or write SEP artifacts. |
| hexagonal-billing | blocked | Claude Code correctly refused to silently auto-approve gates, noted teammate runtime limitations in `claude -p`, and reported that no-edit mode prevents SEP log and ADR artifact writes. |

Conclusion: converting this baseline from contract replay to true live evidence requires an interactive or tmux Claude Code session where gates can be answered and `ai-docs/` artifacts can be written. Non-interactive `claude -p` is useful for probing, but it is not a faithful runner for teammate/gate-heavy workflows.
